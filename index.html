TeoriaVUCA/BANI e os 6 Pilares

🌀 O Mundo Ficou Difícil de Gerenciar

VUCA e BANI descrevem dois olhares sobre a turbulência contemporânea. Em ambos, a vantagem migra para quem decide melhor com dados.

Antifragilidade (Taleb): organizações que se fortalecem com volatilidade usam dados para ver oportunidades nas crises — Amazon, Netflix, Mercado Livre em 2020.

Volatilidade / Brittle

Mudanças rápidas revelam fragilidade estrutural oculta

Incerteza / Ansioso

Dados em excesso ampliam a paralisia decisória

Complexidade / Não-linear

Causas pequenas geram efeitos desproporcionais

A Resposta

Dados encurtam: percepção → interpretação → ação

  • 1. Alinhamento Estratégico — Que decisões precisamos tomar melhor? (BSC, OKRs)
  • 2. Arquitetura de Dados — Mapear fontes, qualidade e lacunas
  • 3. Infraestrutura — Cloud vs. On-Premise; Warehouse vs. Lake
  • 4. Capital Humano — Equipe certa + modelo organizacional
  • 5. Integração Decisória — Dados influenciando decisões reais — o pilar mais raro
  • 6. Monitoramento — Estratégia de dados como sistema vivo

TeoriaVieses e o Ciclo D3

✈️ O Caso Wald — O Dado Invisível

1943: militares propuseram blindar as áreas com mais marcas de bala nos aviões que voltaram. Wald identificou o erro: eram os sobreviventes. As áreas sem marcas eram onde os abatidos foram atingidos.

No negócio: medir NPS só de quem ficou, avaliar campanhas só com retorno, estudar só empresas bem-sucedidas. A ausência de dados é o dado mais importante.

  • Disponibilidade: superestimamos o que está na memória recente → use dados históricos
  • Confirmação: buscamos dados que confirmam o que já cremos → hipótese rival obrigatória
  • Ancoragem: a primeira info ancora todas as avaliações → defina BATNA antes
  • Sobrevivência: vemos só o que chegou até nós → "o que não estou vendo?"
  • Groupthink: grupos suprimem crítica → advocatus diaboli
  • Ilusão de Controle: superestimamos influência em eventos externos
  • Aversão à Perda: dor da perda vale 2× o prazer do ganho → sunk cost não decide o futuro
  • HiPPO Effect: maior salário prevalece sobre dados → pré-mortem e experimentos A/B
  • 1. Definir o Problema — pergunta errada = análise errada
  • 2. Coletar Dados — fontes internas + externas, qualidade e completude
  • 3. Preparar — 60–80% do tempo total. Limpeza e integração
  • 4. Analisar — descritiva, diagnóstica, preditiva ou prescritiva
  • 5. Decidir — insights + julgamento humano. Dados reduzem incerteza, não substituem julgamento
  • 6. Monitorar — loop de aprendizagem retroalimenta projetos futuros

TeoriaGIGO, paradigmas e limites dos modelos

⚠️ GIGO — O Axioma Mais Importante

Garbage In, Garbage Out. Modelos sofisticados com dados incorretos produzem respostas erradas com aparência de precisão — mais perigoso do que não ter análise alguma.

Antes de qualquer análise: de onde vieram esses dados? Quando foram coletados? Há vieses de seleção?

Supervisionado

Dados rotulados. Churn, fraude, crédito, classificação.

Não Supervisionado

Sem rótulo. Segmentação, anomalias, coocorrência.

Por Reforço

Tentativa e erro. Rotas, precificação dinâmica.

4 razões para uma correlação: A causa B | B causa A | C causa ambos | coincidência

Sorvete e afogamentos correlacionam r=0,91. Causa real: temperatura alta no verão.

Antídoto: experimentos A/B randomizados. Amazon e Netflix realizam dezenas de milhares por ano.
